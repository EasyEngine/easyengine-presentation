Understanding the problem

Let's imagine!

WordPress is a car!

And... hosting is road!

This is what you want!

But!!!

Shared hosting!

Cheap hosting!

Overloaded car!

So The Problems are...

  1. With roads... i.e. Hosting/Server
  2. With cars...... i.e. WordPress - Themes/Plugins
  3. Both!

Fixing Hosting problem!

Fixing Server Problem!

ee site create example.com --wp

But there is no easy fix for overloaded site!

Because...

  • Not every site can be fully cached
    • e.g. membeship sites
    • e.g. e-commerce sites
  • Even when full-page cache is present...
    • a missed request will take longer or even fail
    • There is always /wp-admin which cannot be cached

So we need to take harder path...

  1. Trial and error method to find faulty theme/plugin
    • theme/plugin conflicts may get skipped
    • inconvience to the users if site is live
  2. Debug like a hero!

Part 3

Using EasyEngine

EasyEngine Debug Command

MySQL Slow Query Log
ee debug --mysql --import-slow-log

PHP XDebug Profiling
ee debug example.com --php

WordPress Debug Log
ee debug example.com --wp

Install Web-based Tools

Start WordPress debug log
ee stack install --admin
Installs webgrind, anemometer, other tools
