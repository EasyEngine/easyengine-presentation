Part-1

Why Debugging is Needed?

Story so far

  • Full page caching puts zero load on php-mysql backend for cached requests
    • e.g. Nginx fastcgi cache, w3 total cache's disk cache
  • For PHP: use Zend Opcache, HHVM
  • For MySQL: there is query cache

But...

  • Not every site can be cached (full-pgae caching)
    • e.g. membeship sites
    • e.g. e-commerce sites
  • Even when full-page cache is present...
    • If site is coded poorly, a missed request will take longer or even fail.
    • There is always /wp-admin which cannot be cached

Common issues & workarounds

  • Site is running slow or even worse crashing with 5xx server error
    • Increase php/mysql timeout values
    • Add more CPU cores if needed
  • PHP error log is showing out of memory error messages
    • Increase RAM on server

Using EasyEngine

EasyEngine Debug Command

Start mysql slow query log
ee debug --mysql --import-slow-log

Start php (xdebug) profiling
ee debug example.com --php

Start WordPress debug log
ee debug example.com --wp

Install Web-based Tools Command

Start WordPress debug log
ee stack install --admin
